Aide

  • Spécifications complètes du langage
  • Structure générique : 
    models:
    (model 1):
        (champ 1):
	    (paramètres de champ)
        (champ 2):
	    (paramètres de champ)
    (model 2):
        ...
  • Modèles (model n). N'importe quel nom qui ne soit pas un mot clé du langage, par exemple livre. Sinon, le mot doit être précédé d'un point d'exclamation comme !point.
  • Champs (champ n). N'importe quel nom qui ne soit pas un mot clé du langage, par exemple nb_page. Sinon, le mot doit être précédé d'un point d'exclamation. Un champ peut également être le nom d'un autre modèle défini dans le fichier comme page.
  • Paramètres de champ (paramètres de champs).
    Une liste des mots clés suivants : 
    type : "text", "number", "point" ou "file"
    quantity : "{integer}" or "{integer} to {integer}" or "{integer} or {integer}".
    parent : "{integer}" or "{integer} to {integer}" or "{integer} or {integer}".
    constraints :
    pour le type "text" : "null", "max", "min"
    pour le type "number": "positive", "negative", "null", "float", "integer"
    pour les types "text" et "number": "in"
    type vaut "text" par défaut, quantity vaut "1" (un) par défaut et constraints est vide par défaut. Ce qui signifie :
    • Un champ non typé est un texte de longueur indéterminée.
    • Les champs sont obligatoires par défaut. Sinon il faut indiquer quantity: 0 or 1.
    parent n'est valable que si le (champ) est un autre modèle. Il permet de définir la cardinalité de la relation inverse.
